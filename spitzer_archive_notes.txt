NOTES ON SPITZER ARCHIVE:[] The "official" Spitzer archive has 2 or 3 access methods, *none* of which areuseful to us:"Normal" access (and data retrieval) is via Java program Leopard (partof "Spitzer Pride" package, which also includes proposal tool Spot).However, http://archive.spitzer.caltech.edu/also includes a link to a "search by position" page...	http://archive.spitzer.caltech.edu/searchByPositionPage.doRA: "ex: 12h03m22.03228s"Dec: "ex: -0d21m53.8107s"Radius: arcmin (between 1.0 and 90.0)We should probably assume that the RA and Dec *must* be in that particularformat...There is a newer ("beta" page athttp://sha.ipac.caltech.edu/applications/Spitzer/SHA/*However*, both of those web-based "search-by-position" pages use JavaScript,so we're locked out.[] A promising alternat approach is to use the NOAO VO web interface for the "vodata"command-line tool:   http://iraf-nvo.noao.edu/vo-cli/vodata/index.htmlThis allows form-based querying, *if* we specify that we want the "Spitzer" resource (lots ofother services are potentially available).The only problem (aside from minor issues like its use of GET instead of POST) is thatit returns a web page with -- if data are available -- a link to a text file containingmore detailed info.[] Possible backup alternate approach:The Spitzer observing-log table at:http://heasarc.gsfc.nasa.gov/W3Browse/spitzer/spitzmastr.htmlActual web-form interface:http://heasarc.gsfc.nasa.gov/db-perl/W3Browse/w3table.pl?tablehead=name%3Dspitzmastr&Action=More+OptionsThis is in fact the VO "resource" which the NOAO vodata tool makes use of:	ivo://nasa.heasarc/spitzmastr[Currently, as of 15 Sept 2010, this doesn't seem to work]**[] HEASARC
